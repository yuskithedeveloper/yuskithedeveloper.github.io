Technical concept

  • Only HTML \ CSS \ JavaScript

    I want this site to be as simple as possible, both in terms of development and deployment \ hosting. I don't need any IDE \ code editor \ web server \ database \ etc. to work on it. And any hosting will do.

  • Single file

    I like the idea of having anything in one file. It makes fairly easy to store temporary site data. Working with user interface is easier too. Updates are fast and flawless. And no annoying hyperlink navigation flickering.

    On the other hand. There is increased traffic, the need of some kind of JavaScript navigation, the final HTML is bulky.

    But since at this moment the whole site is less that 1MB the concept works fine. For comparison: my LinkedIn page is about 3MB, my recently created and empty GitHub is 2MB, google.com is almost 2MB too... It looks like I have nothing to worry about.

    Obviously, I'll put all the media in separate files (except the site icon and background svg).

  • No third-party

    This site is very simple. It took me just a few days to create it from scratch. It looks not so bad in any of my devices. There is just no need of using any CMS or third-party library.

    Also, this kind of hand-making work brings pleasure:)

  • Any browser \ device friendly

    I test and check that everything works as intended in all browsers and devices at my disposal.

    I even downgraded some JavaScript code, so it can run on older browsers like Internet Explorer.
